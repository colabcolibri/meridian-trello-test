# v5 — Hierarquia relacional agile e UX de produto

## Objective

A v4 entregou cartões e modal alinhados ao template Meridian, mas a gestão de produto continua frágil: versão grava "Included" como lista digitada (`included_json`), épico grava versões em JSON (`version_ids_json`), o hub do projeto mistura abas planas com inputs inline que parecem protótipo, e criar/editar versão, épico ou sprint não oferece fluxo coerente com os wizards de US. O gestor corre risco de IDs órfãos e não enxerga a árvore release → épico → sprint → US. A v5 torna as relações **fonte de verdade no banco** (tabela de junção épico↔versão, FKs já existentes reforçadas na UI) e substitui a experiência de metadata por navegação hierárquica e diálogos de edição completos. "Explicitly out" permanece texto livre na versão; "Included" passa a ser **derivado** das relações, nunca digitado.

## Done criteria

A v5 está completa quando o gestor valida na janela Tauri: criar versão sem campo manual de Included; vincular épico a versão via picker relacional; criar sprint escolhendo versão existente; editar versão/épico/sprint em wizard dedicado (não só blur inline); hub exibe drill-down versão → épicos → sprints → quadro; criar/editar US filtra épico e sprint pela versão selecionada e rejeita combinações incoerentes. Todos Must US v5 ✅ com Record; `validate_meridian.py` ok; desenvolvimento consolidado em `main` (antes branch `agile`).

- Reescrever entregas v4 já ✅ — v5 evolui metadata e navegação sem invalidar cartão/modal US.
