# v3 — Gestão agile de user stories

## Objective

As v1 e v2 entregaram um kanban local de tarefas com polish estilo Trello. O dogfood e a própria governança Meridian mostram que o mesmo usuário precisa organizar **desenvolvimento de software** — não só tarefas genéricas — com épicos, releases, sprints e user stories ricas (Intent, Plan, critérios de aceite, dependências). A v3 pivota o produto: cada **projeto** no SQLite concentra versões, épicos, sprints e US; o quadro kanban passa a representar o fluxo agile das stories (backlog → pronta → em progresso → revisão → concluída), e o modal de detalhe segue o contrato Meridian que já provou robustez na documentação deste repositório. Markdown deixa de ser a fonte de verdade **dentro do app**; o arquivo `kanban.db` passa a persistir o backlog inteiro.

## Done criteria

A v3 está completa quando o gestor valida na janela Tauri: cria um projeto, define versão v1 e sprint v1-S1, cadastra pelo menos três US com critérios de aceite e dependência entre elas, move stories no quadro agile com persistência após reload, edita Why/Where/Acceptance no modal, e filtra o board por versão e épico. Todos os Must US v3 devem estar ✅ com Record preenchido e `validate_meridian.py` sem erros. O modo kanban de tarefas v1/v2 pode coexistir como legado ou ser substituído — decisão registrada no log; o critério de done exige que o fluxo agile seja o caminho principal na UI.

## Explicitly out

- Sincronização com arquivos `.md` do Meridian no disco — import/export fica para versão futura; v3 nasce SQLite-first no app.
- Multi-usuário, permissões, colaboração em tempo real — permanece fora do escopo desde v1.
- Story points, velocity e burndown — Meridian deliberadamente não usa; o app segue a mesma linha.
- Agentes IA ou slash commands dentro do app — o app é ferramenta do gestor humano; governança Meridian continua no repositório/docs.
- Substituir `docs/` deste repositório por SQLite — a documentação Meridian do produto kanban/agile continua em Markdown para desenvolvimento do próprio app.
