# v4 — Cartão e quadro 100% user story Meridian

## Objective

A v3 entregou persistência agile e um quadro paralelo, mas na prática o gestor ainda vê **cartões de tarefa Trello**: título, badges soltos e metadados de prioridade/prazo/checklist no legado `/boards`. A face do cartão agile (`StoryCompact`) não comunica o contrato Meridian — falta o tripé **As / I want / so that**, trecho de **Why**, **done_when**, preview de **Acceptance** e hierarquia visual Intent vs Plan. O modal ajuda, mas quem passa o olho no board não distingue user story de to-do. A v4 redefine o produto: o único cartão do quadro principal é a **US schema v2** (Intent, Plan, Boundaries visíveis ou resumidos na face); estilos `trello-card` de tarefa deixam de ser reutilizados no fluxo agile; `/projects` permanece entrada única e `/boards` vira legado oculto ou redirect.

## Done criteria

A v4 está completa quando o gestor valida na janela Tauri: ao abrir o sprint board, **cada cartão** exibe id US-XXXX, preamble As/I want/so that, linha de Why, done_when, progresso de acceptance e badges épico/versão/MoSCoW/ready/status — sem aparência de cartão de tarefa com labels coloridas de prazo. Criar US exige preamble e done_when, não só título. Modal segue ordem Intent → Plan → Boundaries → Record (read-only). Colunas refletem ciclo Meridian (backlog → refine → ready → em progresso → review → concluída) com regras visuais mínimas. `/` e home abrem projetos agile; kanban v1 não compete na navegação. Todos Must US v4 ✅ com Record; `validate_meridian.py` ok.
