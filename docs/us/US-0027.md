# US-0027 — Back to Boards sem redirect loop

**As** Usuário local,
**I want** voltar à lista de quadros ao clicar "← Boards",
**so that** eu possa trocar de board sem ficar preso no último quadro aberto.

## Intent

### Acceptance

- [x] Link "← Boards" no header do quadro navega para `/` e mostra grid de quadros.
- [x] Não redireciona de volta ao `last_board_id` quando navegação é explícita.
- [x] Auto-resume ao último quadro continua funcionando ao abrir app direto em `/` (cold start).
- [x] `boardService.setLastBoardId` ainda atualizado ao abrir um quadro específico.

### Why

`BoardListPage` sempre redirecionava para `last_board_id` ao montar — incluindo quando o usuário clicava "← Boards". Isso quebrava a navegação workspace e parecia bug crítico.

### Where

v2-S3 / EPIC-10. Depende de US-0006 (last board). Desbloqueia fluxo multi-board.

## Plan

### Approach

- Passar `state: { showWorkspace: true }` no `Link` de `BoardView` para `/`.
- Em `BoardListPage`, ler `useLocation().state.showWorkspace` e pular auto-redirect quando true; carregar lista normalmente.
