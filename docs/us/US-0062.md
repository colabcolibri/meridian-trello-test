# US-0062 — Hub hierárquico versão épico sprint

**As** Usuário local,
**I want** navegar projeto por versão, épico e sprint de forma hierárquica,
**so that** eu encontre o quadro certo e gerencie metadata sem abas confusas.

## Intent

### Acceptance

- [ ] `ProjectHubPage` substitui seção "Product metadata" + abas `ProductCrudPanel` por layout hierárquico: coluna/lista de versões → painel de épicos da versão (`list_epics_for_version`) → sprints da versão → ação "Abrir quadro".
- [ ] Breadcrumb ou header contextual: `Projeto / vX / [épico opcional] / sprint`.
- [ ] Botões criar versão, épico (pré-seleciona versão ativa), sprint (pré-seleciona versão ativa) no contexto correto — abrem wizards de US-0059/0060/0061.
- [ ] Included derivado visível no painel da versão selecionada (read-only summary).
- [ ] Layout responsivo mobile/tablet/desktop sem overflow horizontal (`us-form.css` / trello tokens).
- [ ] `ProductCrudPanel` removido ou reduzido a shim deprecated — hub é caminho principal.

### Why

A experiência atual mistura Navigation (dois selects) com Product metadata (três abas planas e inline edit) — gestor descreve como "horrível". Com CRUD wizards prontos (US-0059–0061), este slice reorganiza a superfície em árvore alinhada ao modelo mental Meridian e elimina duplicação de navegação.

### Where

v5-S3 / EPIC-19. Depende dos wizards de versão, épico e sprint. Unblocks validação end-to-end antes US-0063. Não altera lógica cascata nos forms de US.

## Plan

### Approach

- Refatorar `ProjectHubPage.tsx`: estado `selectedVersionId`, `selectedEpicId?`, `selectedSprintId`; carregar épicos via `epicService.listForVersion` e sprints filtrados por `version_id` — reutilizar filtros já usados em `CreateUsDialog` filteredSprints pattern.
- Criar componentes presentacionais `VersionListPanel`, `EpicListPanel`, `SprintListPanel` em `src/features/agile/ui/hub/` — SRP UI; recebem props e callbacks onEdit/onCreate/onOpenBoard.
- Integrar `VersionIncludedSummary`, `EditVersionDialog`, `EditEpicDialog`, `EditSprintDialog`, dialogs create existentes — um mapa de modals no hub, não duplicar em ProductCrudPanel.
- CSS: grid `hub-layout` em `us-form.css` ou `trello-theme.css` com breakpoints `md:` stacks columns; cards clicáveis com estado selected.
- Remover import `ProductCrudPanel` do hub ou deixar link "Modo lista (legado)" collapsed — preferir remoção se paridade atingida.
