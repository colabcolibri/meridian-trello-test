# US-0003 — Commands Tauri e tipos de domínio

**As** Usuário local,
**I want** que operações de quadro sejam expostas via API tipada entre UI e SQLite,
**so that** features possam persistir dados sem duplicar lógica SQL no React.

## Intent

### Acceptance

- [x] Tipos `Board`, `Column`, `Card`, `Tag`, `ChecklistItem` em `src/domain/types.ts`.
- [x] Commands mínimos implementados: `list_boards`, `create_board`, `update_board`, `delete_board`, `get_last_board_id`, `set_last_board_id`.
- [x] Hook ou serviço `src/features/boards/` invoca commands com tratamento de erro básico.

### Why

UI não deve falar SQL. Esta US fecha a fundação de persistência para quadros — prova o padrão invoke + domain types que colunas e cartões reutilizarão.

### Where

Final de v1-S1 / EPIC-01. Depende de US-0002. Desbloqueia US-0004 (shell) e US-0005 (CRUD UI quadros).

## Plan

### Approach

- Definir tipos puros em `src/domain/types.ts` (`Board`, `Column`, `Card`, `Tag`, `ChecklistItem`) e validador `validateBoardName` em `src/domain/validators.ts`, reutilizados por todas as features — única fonte de regra de nome não vazio.
- Implementar commands Rust em `src-tauri/src/commands/boards.rs` (`list_boards`, `create_board`, `update_board`, `delete_board`, `get_last_board_id`, `set_last_board_id`) registrados em `lib.rs`, usando transações e `with_connection` de US-0002.
- Criar `src/features/boards/boardService.ts` como único wrapper de `invoke` para boards; página de teste mínima ou DevTools console prova o ciclo antes da UI de US-0005.
