# US-0023 — Colunas e header do quadro estilo Trello

**As** Usuário local,
**I want** colunas e cabeçalho do quadro organizados como no Trello,
**so that** eu navegue o pipeline com familiaridade imediata.

## Intent

### Acceptance

- [x] Header do board: nome proeminente, navegação “Quadros” discreta, área de ações alinhada (sem inputs cruas de protótipo).
- [x] Coluna: fundo semi-transparente/claro sobre board, título uppercase ou semibold compacto, contador opcional de cartões.
- [x] Botão “Adicionar um cartão” estilo Trello (texto + hover) substitui input sempre visível como padrão; input expande ao clicar.
- [x] Scroll horizontal apenas na faixa de colunas; header fixo.
- [x] Ações de coluna (renomear, mover, excluir) acessíveis por menu ⋯, não emoji solto.

### Why

Colunas são o container principal do fluxo kanban. O layout atual expõe controles técnicos (setas, lixeira emoji, input permanente) que quebram a ilusão de produto acabado. Esta US aplica tokens de US-0022 ao chrome estrutural do board.

### Where

v2-S1 / EPIC-09. Depende de US-0022 (tokens). Desbloqueia polish final em US-0024 (cartões dentro das colunas).

## Plan

### Approach

- Refatorado `ColumnPanel` em `BoardView.tsx` para usar classes `.trello-column`, contador de cartões no título e menu ⋯ com renomear/mover/excluir — removidos botões emoji ▲▼🗑 da v1.
- Header do board migrado para `.trello-board-header` semi-transparente sobre gradiente; botão “＋ Adicionar coluna” expande input inline em vez de campo sempre visível.
- Botão “＋ Adicionar um cartão” expande textarea estilo Trello; scroll horizontal restrito à faixa `.kanban-scroll`.
