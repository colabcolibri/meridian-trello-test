# US-0031 — Criar label inline e aplicar ao cartão

**As** Usuário local,
**I want** criar uma label pelo modal e vê-la no cartão imediatamente,
**so that** eu categorize tarefas sem fluxo quebrado.

## Intent

### Acceptance

- [x] "＋ New label" abre formulário inline (nome + cor) — não usa `window.prompt`.
- [x] Ao confirmar, tag é criada no board **e** associada ao cartão via `set_card_tags`.
- [x] Label aparece ativa no modal e no `CardCompact` após fechar modal.
- [x] Erros de validação/rede exibidos no formulário.

### Why

Dogfood reportou "New label" sem efeito. Causa dupla: (1) `window.prompt` não funciona de forma confiável no webview Tauri; (2) `create_tag` só adicionava tag ao board sem chamar `setCardTags` — cartão permanecia sem label.

### Where

v2-S4 / EPIC-10. Depende de US-0014 (tags). Fecha item da auditoria v2-S3 sobre `window.prompt`.

## Plan

### Approach

- Substituir `window.prompt` por state `showNewLabel` + input + color picker em `CardDetailModal.tsx`.
- Após `create_tag`, chamar `setCardTags` com ids existentes + novo id; atualizar `detail.tags`, `boardTags` e `onChanged()`.
