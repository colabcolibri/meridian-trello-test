# US-0065 — Banner demo, base Vite e deploy GitHub Pages

**As** visitante que abre o link público da demo,
**I want** entender que meus dados ficam só no navegador e acessar o app no subpath do GitHub Pages,
**so that** confio no modo demo e os assets carregam corretamente.

## Intent

### Acceptance

- [x] `BrowserDemoBanner` shows local-only notice and “Clear demo data” button (confirm → `clearBrowserAgileStore` + reload); hidden in Tauri runtime.
- [x] `vite.config.ts` usa `base: process.env.VITE_BASE_PATH ?? "/"`; `router.tsx` usa `basename={import.meta.env.BASE_URL}`.
- [x] Workflow `.github/workflows/deploy-pages.yml` na branch `agile` builda com `VITE_BASE_PATH: /meridian-trello-test/` e publica artifact Pages.
- [x] Estilos responsivos do banner em `us-form.css` — sem overflow horizontal em mobile.

### Why

Sem banner, visitantes podem achar que dados sobem ao servidor; sem `base` correto, GitHub Pages quebra rotas e assets. Este slice fecha UX de transparência e pipeline estático para compartilhar a demo v6.

### Where

v6-S2 / EPIC-20. Depende US-0064 (store operacional). Completa Must de v6 junto com EPIC-20.

## Plan

### Approach

- Componente `BrowserDemoBanner` montado via `TauriGuard` — condicional `isTauriRuntime()` evita ruído no desktop.
- Configurar Vite `base` via env no CI e local default `/` para dev — DRY com `import.meta.env.BASE_URL` no router.
- Workflow Pages com permissões `pages: write` + `deploy-pages@v4`; path artifact `dist`.
