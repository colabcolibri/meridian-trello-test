# US-0026 — Corrigir modal de cartão get_card JSON

**As** Usuário local,
**I want** abrir o detalhe de um cartão ao clicar nele,
**so that** eu edite título, descrição e metadados sem a tela quebrar.

## Intent

### Acceptance

- [x] Clicar em cartão na coluna abre modal com título e campos visíveis.
- [x] Sem `TypeError: undefined is not an object (evaluating 'card.title')` no console.
- [x] Resposta de `get_card` alinha com tipo `CardDetail` do frontend (`{ card, tags, checklist }`).
- [x] Falha de carregamento exibe mensagem e botão fechar — não tela em branco.

### Why

Dogfood pós-v2 revelou regressão: `CardDetail` no Rust usava `#[serde(flatten)]` no campo `card`, serializando campos do cartão na raiz do JSON. O frontend espera objeto aninhado `detail.card`; ao renderizar, `card` era `undefined` e o React crashava — usuário via overlay escuro sem conteúdo.

### Where

v2-S2 / EPIC-08 (correções). Depende de US-0025 (modal Trello). Desbloqueia uso normal do fluxo kanban.

## Plan

### Approach

- Remover `#[serde(flatten)]` de `CardDetail` em `src-tauri/src/models.rs` para que `get_card` retorne `{ card: {...}, tags: [...], checklist: [...] }` conforme `docs/07_api_contracts.md` e `src/domain/types.ts`.
- Adicionar tratamento de erro em `CardDetailModal.tsx` no `load()`: validar `d.card`, exibir mensagem amigável se invoke falhar ou payload for inválido.

- `docs/architecture/ui-kanban.md` — § Modal de detalhes
