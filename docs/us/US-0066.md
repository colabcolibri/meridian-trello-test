# US-0066 — Reset demo e rotas stale no browser

**As** visitante da demo web,
**I want** resetar a demo e voltar sempre à lista de projetos,
**so that** não fique preso em URL com id que não existe mais.

## Intent

### Acceptance

- [x] `resetBrowserDemoAndGoToProjects()` restaura seed e navega para `{BASE_URL}projects` (banner usa isso no confirm).
- [x] `ProjectHubPage` distingue loading vs erro: projectId inválido exibe mensagem + link “Back to projects”, não spinner infinito.
- [x] `ProjectListPage` no browser demo não auto-redireciona para last project (visitante vê lista após reset).
- [x] `pnpm exec tsc --noEmit` ok.

### Why

Após reset, localStorage só tem o projeto demo fixo; URL antiga com UUID gerado quebra `get_project` e a UI ficava em “Carregando projeto…” para sempre.

### Where

v6-S3 / EPIC-20. Depende US-0064. Complementa US-0065 (banner).

## Plan

### Approach

- Centralizar navegação pós-reset em `src/lib/browserDemoChrome.ts` com `demoProjectsUrl()` respeitando `import.meta.env.BASE_URL`.
- Refatorar `load` do hub: estado `loading`, catch em `get_project`, remover `selectedVersionId` das deps do callback (evita re-fetch desnecessário).
