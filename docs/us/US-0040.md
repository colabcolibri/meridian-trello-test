# US-0040 — Filtros por versão, épico e sprint no quadro

**As** Usuário local,
**I want** filtrar o quadro por versão, épico ou sprint,
**so that** eu foque no release ou capability certo durante planning.

## Intent

### Acceptance

- [x] Barra acima do board com selects: Versão (All + cada vX), Épico (All + EPIC-XX), Sprint (All + sprints da versão selecionada).
- [x] Filtros combinam (AND); alterar versão não zera épico se ainda válido.
- [x] Contador "N stories" reflete filtro ativo.
- [x] Estado de filtro persiste na sessão (context ou preferences) ao trocar de tela dentro do projeto.

### Why

Sprints podem compartilhar backlog de versão; épicos cruzam sprints. Sem filtros, o quadro mistura ruído — mesmo problema que o writing-guide descreve para monitor Meridian com v0 e v1 juntos. Esta slice é Should porque o board mínimo (US-0038) já funciona no sprint ativo; filtros ampliam foco cross-sprint.

### Where
