# US-0061 — CRUD sprint com wizard de edição

**As** Usuário local,
**I want** editar sprints completos incluindo goal e vínculo à versão,
**so that** eu não fique preso a renomear sprint por blur inline na lista.

## Intent

### Acceptance

- [ ] Novo `EditSprintDialog` reutiliza steps de `CreateSprintDialog` (meta, goal, scope, review) pré-preenchidos; versão exibida como select desabilitado ou read-only (alterar versão de sprint existente fora de escopo).
- [ ] `ProductCrudPanel` / hub oferece Editar sprint abrindo wizard — não blur inline em goal/done_when.
- [ ] `CreateSprintDialog` continua exigindo versão via select populado de `versionService.list` — já relacional; melhorar labels `{id} — {title}` e empty state se zero versões.
- [ ] Alterações em goal, done_when, out_of_scope, title, status persistem após reload.
- [ ] Lista de sprints agrupa ou filtra por versão selecionada no contexto pai (prepara US-0062).

### Why

Sprint já tem FK `version_id` no SQLite — melhor relação do trio — mas a UI trata sprint como linha genérica editável só no título. Gestor não consegue revisar goal ou done_when após criação sem hacks. Este slice entrega paridade de wizard com versão/épico.

### Where

v5-S3 / EPIC-19. Depende US-0058 apenas para consistência de projeto; FK sprint→versão já existia. Unblocks US-0062 (hub abre board a partir de sprint na árvore). Não altera US forms.

## Plan

### Approach

- Criar `EditSprintDialog.tsx` copiando estrutura de `CreateSprintDialog.tsx` com prop `sprintId`; load via `sprintService.get` ou list+find; submit `sprintService.update` — reutilizar STEPS array exportado ou constante shared para DRY.
- Refatorar `ProductCrudPanel` tab sprints: row com versão badge, botão Editar, remover onBlur em goal implícito (hoje só title blur).
- Opcional shared `SprintFormSteps` entre create/edit em `src/features/agile/ui/sprintFormShared.tsx` se duplicação > 30 linhas — respeitar SRP (só UI sprint).
- Garantir `sprintService.ts` expose get by id se ausente — thin invoke wrapper.
