# US-0005 — CRUD de quadros

**As** Usuário local,
**I want** gerenciar meus quadros de projetos,
**so that** eu separe áreas de trabalho distintas no mesmo app.

## Intent

### Acceptance

- [x] Criar quadro com nome; aparece na lista imediatamente.
- [x] Renomear quadro inline ou via dialog; nome persiste após reload.
- [x] Selecionar quadro navega para visão do quadro (`/boards/:id`).
- [x] Excluir quadro pede confirmação; remove board e dados cascata (colunas/cartões).

### Why

Quadros são o container raiz do kanban. Sem CRUD, o usuário ficaria preso a um único board hardcoded. Esta US entrega gestão completa de projetos locais.

### Where

EPIC-02, v1-S2. Depende de US-0004. Desbloqueia US-0006 e US-0007 (colunas dentro do quadro).

## Plan

### Approach

- Estender `boardService` com métodos `updateBoard` e `deleteBoard` mapeando commands já definidos em `07_api_contracts.md`, reutilizando validação de `validators.ts` — sem duplicar regra de nome.
- Implementar `src/features/boards/ui/BoardListPage.tsx` com grid/lista de `BoardCard`, botão "Novo quadro" abrindo `CreateBoardDialog`, e rename inline no card ou dialog — navegação para `/boards/:id` ao selecionar.
- Adicionar `DeleteBoardConfirm` modal antes de `deleteBoard`; cascade no SQLite (US-0002) remove colunas/cartões — UI só confirma intenção.
