# US-0021 — Corrigir DnD entre colunas sem clone

**As** Usuário local,
**I want** arrastar cartões entre colunas sem ver duplicatas,
**so that** eu confie que o kanban reflete exatamente o andamento das tarefas.

## Intent

### Acceptance

- [x] Arrastar cartão da coluna A para B remove o item da origem e posiciona na coluna destino — nunca dois cards com o mesmo id visíveis.
- [x] Soltar em área vazia da coluna destino appenda o cartão ao final.
- [x] Reordenar dentro da mesma coluna continua funcionando.
- [x] Ordem e column_id persistem após fechar e reabrir o app.
- [x] Cancelar arraste (Esc ou soltar fora) restaura layout anterior.
- [x] Falha de persistência reverte UI para estado do servidor.

### Why

A v1 marcou US-0018 como entregue, mas dogfood revelou regressão: ao mover entre colunas o cartão "clona" visualmente para baixo em vez de transferir. Isso invalida o gesto principal do produto e precisa correção antes de investir em polish visual.

### Where

v2-S1 / EPIC-08. Depende dos commands `move_card` e `reorder_cards` de US-0011/US-0018. Desbloqueia US-0024 (feedback visual de arraste integrado ao design Trello).

## Plan

### Approach

- Substituído o `onDragEnd` exclusivo por `onDragOver` com lógica de multi-container: ao detectar que o cartão cruzou para outra coluna, o estado local é atualizado imediatamente (remoção da origem + inserção no destino), eliminando o clone visual.
- Adicionado `useDroppable` por coluna e `dragStartColumnRef` para rastrear a coluna de origem durante o arraste; `onDragCancel` restaura o snapshot salvo no início do gesto, garantindo reversão limpa ao pressionar Esc ou soltar fora de qualquer droppable.
- Placeholder tracejado (`trello-drag-placeholder`) inserido na posição original enquanto o arraste está ativo, mantendo o layout estável e comunicando ao usuário o ponto de retorno.
