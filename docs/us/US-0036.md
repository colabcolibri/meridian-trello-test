# US-0036 — Workspace: listar e criar projetos

**As** Usuário local,
**I want** ver meus projetos ao abrir o app e criar um novo rapidamente,
**so that** eu escolha em qual produto trabalhar hoje.

## Intent

### Acceptance

- [x] Tela workspace (ou modo agile na home) lista projetos via `list_projects` com nome e data.
- [x] Botão "Novo projeto" abre formulário inline ou modal — sem `window.prompt`.
- [x] Após criar, projeto aparece na lista e pode ser selecionado como contexto ativo (preferência em `app_preferences`).
- [x] Layout responsivo 1024px+ alinhado ao design system Trello v2.

### Why

A API de projetos (US-0033) não ajuda o gestor sem superfície visual. Esta slice substitui ou complementa a lista de "boards" como entrada principal para fluxo agile — sem ainda mostrar quadro de US. O usuário passa a pensar em "projetos de software", não só quadros de tarefa.

### Where

v3-S2 / EPIC-12. Depende de US-0033. Desbloqueia US-0037 (navegação interna).
