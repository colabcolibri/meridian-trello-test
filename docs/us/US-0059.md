# US-0059 — CRUD versão com Included derivado

**As** Usuário local,
**I want** criar e editar versões sem digitar épicos incluídos,
**so that** o escopo da release reflita automaticamente o que está ligado no banco.

## Intent

### Acceptance

- [ ] `CreateVersionDialog` remove `StringListEditor` de "Included"; passo Scope exibe apenas "Explicitly out" (textarea manual) e bloco read-only "Included (derivado)" chamando `get_version_included` após vínculos existirem (vazio na criação).
- [ ] Novo `EditVersionDialog` reutiliza `MeridianWizardShell` com todos os campos de versão (objective, done_criteria, explicitly_out, go_live) — equivalente ao create.
- [ ] `ProductCrudPanel` (ou substituto parcial) oferece botão "Editar" por versão abrindo `EditVersionDialog` em vez de depender só de blur inline no título.
- [ ] Seção Included derivada lista épicos de `version_epics` com id + title; após reload persiste coerente com vínculos de épico.
- [ ] Validação de create não exige mais linhas em Included manual.

### Why

O gestor hoje digita "EPIC-01 — …" na versão enquanto o épico marca versões em checkbox — duas fontes que divergem. Este slice remove o input manual de Included, mantém Explicitly out como texto livre (decisão de produto), e entrega edição completa da versão que hoje é praticamente impossível além do título.

### Where

v5-S2 / EPIC-19. Depende de US-0058 (`get_version_included`, junction). Unblocks US-0062 (hub mostra Included derivado no contexto da versão). Não altera épico/sprint neste slice.

## Plan

### Approach

- Refatorar `CreateVersionDialog.tsx`: remover estado `included` e validação que exige `StringListEditor`; adicionar componente `VersionIncludedSummary` em `src/features/agile/ui/` que chama `versionService.getIncluded(projectId, versionId)` — reutilizar estilos `us-review-card` de `us-form.css`.
- Criar `EditVersionDialog.tsx` espelhando steps de create; carregar versão via `versionService.get` ou list+find; submit chama `versionService.update` sem `includedJson`.
- Atualizar `versionService.ts` para omitir `included_json` no payload create/update e expor `getIncluded` tipado com `VersionIncluded` de US-0058.
- Em `ProductCrudPanel.tsx`, substituir edição inline de campos longos por row com título, status badge e botões Editar/Abrir — blur inline só para rename rápido de title se desejado, ou remover blur de metadata longa.
