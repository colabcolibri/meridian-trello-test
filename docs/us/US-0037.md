# US-0037 — Navegação projeto → versão → sprint

**As** Usuário local,
**I want** navegar da lista de projetos até o sprint em que estou trabalhando,
**so that** eu sempre saiba em qual release e timebox estou.

## Intent

### Acceptance

- [x] Após selecionar projeto, UI mostra versões (`list_versions`) com status badge.
- [x] Selecionar versão mostra sprints (`list_sprints`) — sprint `active` destacado se existir.
- [x] Selecionar sprint define contexto global (React context ou route) usado pelo board US-0038.
- [x] "← Projetos" retorna ao workspace sem loop de auto-resume indevido.
- [x] Último project_id / sprint_id persistido em preferences.

### Why

Hierarquia agile não cabe em uma tela única: o gestor precisa de orientação espacial projeto → release → sprint, espelhando `/plan-sprint` no Meridian. Esta slice conecta workspace (US-0036) aos dados de US-0034 sem implementar o kanban ainda.

### Where

v3-S2 / EPIC-12. Depende de US-0036 e US-0034. Desbloqueia US-0038 e US-0043.
