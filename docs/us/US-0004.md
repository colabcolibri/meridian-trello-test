# US-0004 — Shell app rota inicial workspace

**As** Usuário local,
**I want** abrir o app direto no ambiente de trabalho,
**so that** eu comece a organizar tarefas sem telas intermediárias.

## Intent

### Acceptance

- [x] Rota `/` renderiza workspace — lista de quadros ou área vazia com CTA criar quadro.
- [x] Não existe landing page ou fluxo de login.
- [x] Layout shell inclui header mínimo e área de conteúdo responsiva.

### Why

A spec exige primeira tela = trabalho, não marketing. Esta US entrega navegação e shell visual antes do CRUD completo de quadros.

### Where

v1-S2 / EPIC-02. Depende de US-0003. Desbloqueia US-0005 e US-0006.

## Plan

### Approach

- Configurar React Router v7 em `src/app/router.tsx` com rotas `/` (workspace) e `/boards/:boardId` (quadro), montado em `src/app/main.tsx`, para cumprir spec de primeira tela = trabalho sem landing.
- Criar `src/app/layouts/WorkspaceLayout.tsx` com header mínimo (título app, slot de ações) e área de conteúdo responsiva — componentes filhos recebem props, sem invoke direto (SRP UI).
- Implementar `src/features/boards/ui/WorkspaceHome.tsx` listando boards via `boardService.listBoards()` ou empty state com CTA "Criar quadro"; redirect para `/boards/:id` fica para US-0006.
