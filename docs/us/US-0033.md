# US-0033 — Commands CRUD de projetos

**As** Usuário local,
**I want** criar e gerenciar projetos de software no app,
**so that** eu separe backlogs de produtos diferentes no mesmo SQLite.

## Intent

### Acceptance

- [x] Commands: `list_projects`, `create_project`, `update_project`, `delete_project` (com cascade ou bloqueio documentado).
- [x] Projeto tem id, name, description opcional, created_at, updated_at.
- [x] Delete exige confirmação no caller ou retorna erro se projeto tem versões — regra documentada em `07_api_contracts.md`.
- [x] Tipos TypeScript espelham payloads em `src/domain/` ou feature agile.

### Why

Projeto é a raiz da hierarquia agile: sem CRUD estável, workspace e board não têm contexto. Esta slice cobre apenas projetos — versões e sprints ficam na US-0034. Após implementação, testes manuais via invoke ou hook temporário provam persistência.

### Where

v3-S1 / EPIC-11. Depende de US-0032 (schema). Desbloqueia US-0036 (workspace UI).

## Plan
