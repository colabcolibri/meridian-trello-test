# US-0006 — Persistir último quadro usado

**As** Usuário local,
**I want** retomar automaticamente o último quadro que usei,
**so that** eu continue de onde parei sem navegar toda vez.

## Intent

### Acceptance

- [x] Ao abrir quadro, `set_last_board_id` grava preferência em `app_preferences`.
- [x] Na próxima abertura do app, se id válido → redirect para `/boards/:id`.
- [x] Se último quadro foi excluído → fallback para lista de quadros.

### Why

Fluxo "abrir e trabalhar" é requisito explícito da spec. Esta US elimina fricção de seleção repetida — complementa US-0005 sem adicionar novas entidades.

### Where

Fim v1-S2 / EPIC-02. Depende de US-0005. Desbloqueia US-0007 (view colunas do quadro).

## Plan

### Approach

- Chamar `boardService.setLastBoardId(id)` no `useEffect` ou handler de navegação quando usuário entra em `/boards/:boardId`, centralizando preferência num único ponto — DRY com commands já expostos em US-0003.
- No bootstrap de `src/app/App.tsx` (ou loader de rota `/`), invocar `getLastBoardId`; se UUID válido e board existe, `navigate(/boards/:id)`; senão permanecer em `/` — evita loop se board foi excluído.
- Tratar board excluído: se `getBoard` falhar após redirect, limpar preferência via `setLastBoardId(null)` e redirecionar à lista.
