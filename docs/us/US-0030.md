# US-0030 — Drag handle separado do clique no cartão

**As** Usuário local,
**I want** clicar no cartão para editar e arrastar só pelo handle,
**so that** abrir o modal não conflite com drag-and-drop.

## Intent

### Acceptance

- [x] Clique no corpo do cartão abre modal de detalhe de forma confiável.
- [x] Arraste inicia apenas pelo handle `⠿` (visível no hover).
- [x] `setActivatorNodeRef` do `@dnd-kit/sortable` no handle — padrão recomendado.
- [x] DnD cross-column continua funcionando após mudança.

### Why

Listeners de drag cobriam o card inteiro incluindo o botão `CardCompact` — cliques falhavam ou competiam com gesto de arraste, especialmente após US-0021.

### Where

v2-S3 / EPIC-10. Depende de US-0021 DnD estável.

## Plan

### Approach

- Em `SortableCard` (`BoardView.tsx`), mover `{...listeners}` para botão handle com `setActivatorNodeRef`; `setNodeRef` permanece no wrapper.
